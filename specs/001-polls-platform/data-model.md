# Data Model: Polls Platform

**Branch**: `001-polls-platform` | **Phase**: 1 | **Date**: 2026-05-08

## Overview

The data model is **fully implemented** in [`backend/prisma/schema.prisma`](../../backend/prisma/schema.prisma) and the first migration has been applied. This document describes the canonical model as it exists, the validation rules enforced at each layer, and the state transitions relevant to the feature.

No schema changes are required for this feature. All implementation work is frontend-only.

### Poll

**Table**: `polls` | **Source**: [`schema.prisma:44`](../../backend/prisma/schema.prisma:44)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` (UUID) | PK, auto-generated | |
| `title` | `String` | NOT NULL | Used to generate `slug` |
| `description` | `String?` | nullable | Optional |
| `slug` | `String` | UNIQUE, NOT NULL | URL-safe identifier; generated from title |
| `accessToken` | `String` | UNIQUE, NOT NULL, default uuid | Bearer token for PRIVATE polls |
| `visibility` | `Visibility` enum | NOT NULL, default `PUBLIC` | `PUBLIC` or `PRIVATE` |
| `isActive` | `Boolean` | NOT NULL, default `true` | Inactive polls reject submissions |
| `expiresAt` | `DateTime?` | nullable | Submissions rejected after this time |
| `ownerId` | `String` (FK) | NOT NULL | References `users.id`, CASCADE delete |
| `createdAt` | `DateTime` | NOT NULL, default now | |
| `updatedAt` | `DateTime` | NOT NULL, auto-updated | |

**Relations**: belongs to one `User` (owner); has many `Question`s; has many `Response`s.

**Validation rules**:
- `title`: non-empty string
- `questions`: 1–20 questions per poll (FR-007)
- `slug`: unique; generated by [`generateSlug(title)`](../../backend/src/common/utils/slugify.ts) with collision suffix (FR-009)
- `accessToken`: regenerated via `POST /api/v1/polls/:slug/regenerate-token` (FR-014)

**State transitions**:
```
isActive: true  ──(owner deactivates)──►  isActive: false
isActive: false ──(owner activates)────►  isActive: true
expiresAt: null ──(owner sets date)────►  expiresAt: DateTime
expiresAt: past ──────────────────────►  submissions rejected (GoneException)
```

### Question

**Table**: `questions` | **Source**: [`schema.prisma:64`](../../backend/prisma/schema.prisma:64)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` (UUID) | PK, auto-generated | |
| `pollId` | `String` (FK) | NOT NULL | References `polls.id`, CASCADE delete |
| `text` | `String` | NOT NULL | Question prompt |
| `type` | `QuestionType` enum | NOT NULL | `SINGLE_CHOICE`, `MULTIPLE_CHOICE`, or `TEXT` |
| `orderIndex` | `Int` | NOT NULL | Display order (0-based) |
| `isRequired` | `Boolean` | NOT NULL, default `true` | Required questions block submission if unanswered |
| `createdAt` | `DateTime` | NOT NULL, default now | |

**Relations**: belongs to one `Poll`; has many `Option`s (for choice types); has many `Answer`s.

**Validation rules**:
- `type = SINGLE_CHOICE` or `MULTIPLE_CHOICE`: must have at least 1 `Option`
- `type = TEXT`: `options` array must be empty
- Max 20 questions per poll (enforced in `ReplaceQuestionsDto` and `CreatePollDto`)

**Cascade behaviour**: Deleting a `Poll` cascades to delete all its `Question`s, `Option`s, and `Answer`s. Replacing questions (`PUT /api/v1/polls/:slug/questions`) deletes all existing questions first, then recreates them — this also deletes all collected `Answer`s (FR-011).

### Response

**Table**: `responses` | **Source**: [`schema.prisma:92`](../../backend/prisma/schema.prisma:92)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` (UUID) | PK, auto-generated | |
| `pollId` | `String` (FK) | NOT NULL | References `polls.id`, CASCADE delete |
| `userId` | `String?` (FK) | nullable | References `users.id`, SET NULL on delete |
| `respondentFingerprint` | `String?` | nullable | UUID from `localStorage`; soft dedup for anon users |
| `submittedAt` | `DateTime` | NOT NULL, default now | |

**Unique constraint**: `(pollId, userId)` — prevents duplicate authenticated responses (FR-005).

**Relations**: belongs to one `Poll`; optionally belongs to one `User`; has many `Answer`s.

**Deduplication logic** (enforced in [`ResponsesService.submit()`](../../backend/src/responses/responses.service.ts:17)):
- Authenticated user: hard dedup via `(pollId, userId)` unique constraint → `ConflictException`
- Anonymous user: soft dedup via `respondentFingerprint` lookup → `ConflictException` (bypassable by clearing localStorage)
- Frontend: cookie `responded_<slug>=true` provides instant UI-level dedup (FR-004)

### Answer

**Table**: `answers` | **Source**: [`schema.prisma:107`](../../backend/prisma/schema.prisma:107)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` (UUID) | PK, auto-generated | |
| `responseId` | `String` (FK) | NOT NULL | References `responses.id`, CASCADE delete |
| `questionId` | `String` (FK) | NOT NULL | References `questions.id`, CASCADE delete |
| `optionId` | `String?` (FK) | nullable | References `options.id`, SET NULL on delete |
| `textValue` | `String?` | nullable | Free-text answer |

**Invariants**:
- `SINGLE_CHOICE` question: exactly 1 `Answer` record with `optionId` set, `textValue` null
- `MULTIPLE_CHOICE` question: N `Answer` records (one per selected option), each with `optionId` set
- `TEXT` question: exactly 1 `Answer` record with `textValue` set, `optionId` null

### RefreshToken

**Table**: `refresh_tokens` | **Source**: [`schema.prisma:121`](../../backend/prisma/schema.prisma:121)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` (UUID) | PK, auto-generated | |
| `userId` | `String` (FK) | NOT NULL | References `users.id`, CASCADE delete |
| `tokenHash` | `String` | UNIQUE, NOT NULL | bcrypt hash of the raw token |
| `expiresAt` | `DateTime` | NOT NULL | Token TTL |
| `createdAt` | `DateTime` | NOT NULL, default now | |

**Security**: Raw token is sent to the client in an HTTP-only cookie. Only the hash is stored. Tokens are rotated on use (old hash deleted, new hash inserted).

## Entity Relationship Diagram

```
User (users)
 ├── id (PK)
 ├── email (UNIQUE)
 ├── passwordHash
 ├── name
 ├── role: USER | ADMIN
 ├── createdAt, updatedAt
 │
 ├──< Poll (polls) [ownerId → users.id CASCADE]
 │    ├── id (PK)
 │    ├── title, description?
 │    ├── slug (UNIQUE)
 │    ├── accessToken (UNIQUE)
 │    ├── visibility: PUBLIC | PRIVATE
 │    ├── isActive, expiresAt?
 │    ├── ownerId (FK → users.id)
 │    ├── createdAt, updatedAt
 │    │
 │    ├──< Question (questions) [pollId → polls.id CASCADE]
 │    │    ├── id (PK)
 │    │    ├── text, type: SINGLE_CHOICE | MULTIPLE_CHOICE | TEXT
 │    │    ├── orderIndex, isRequired
 │    │    │
 │    │    └──< Option (options) [questionId → questions.id CASCADE]
 │    │         ├── id (PK)
 │    │         ├── text, orderIndex
 │    │         └── (referenced by Answer.optionId)
 │    │
 │    └──< Response (responses) [pollId → polls.id CASCADE]
 │         ├── id (PK)
 │         ├── pollId (FK → polls.id)
 │         ├── userId? (FK → users.id SET NULL)
 │         ├── respondentFingerprint?
 │         ├── submittedAt
 │         ├── UNIQUE(pollId, userId)
 │         │
 │         └──< Answer (answers) [responseId → responses.id CASCADE]
 │              ├── id (PK)
 │              ├── responseId (FK → responses.id)
 │              ├── questionId (FK → questions.id)
 │              ├── optionId? (FK → options.id SET NULL)
 │              └── textValue?
 │
 └──< RefreshToken (refresh_tokens) [userId → users.id CASCADE]
      ├── id (PK)
      ├── tokenHash (UNIQUE)
      └── expiresAt
```

## Frontend Data Contracts (TypeScript)

These types are used by the frontend to consume the API. They live in `frontend/src/types/`.

### `Poll` (list item)

```typescript
// frontend/src/types/poll.ts
interface PollSummary {
  id: string;
  title: string;
  slug: string;
  description: string | null;
  visibility: 'PUBLIC' | 'PRIVATE';
  isActive: boolean;
  expiresAt: string | null;  // ISO 8601
  responseCount: number;
  owner: { id: string; name: string };
  createdAt: string;
}
```

### `Poll` (detail — for poll-taking page)

```typescript
interface PollDetail extends PollSummary {
  questions: Question[];
  updatedAt: string;
}

interface Question {
  id: string;
  text: string;
  type: 'SINGLE_CHOICE' | 'MULTIPLE_CHOICE' | 'TEXT';
  orderIndex: number;
  isRequired: boolean;
  options: Option[];
}

interface Option {
  id: string;
  text: string;
  orderIndex: number;
}
```

### Response submission payload

```typescript
interface SubmitResponsePayload {

interface AnswerPayload {
```
